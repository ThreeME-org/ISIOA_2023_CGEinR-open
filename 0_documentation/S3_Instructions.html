Session 3: Running the Walras and Neo Keynesian model

This session is in two parts: building on what we have learned regarding running a simple model, we now look into more complex (and more interesting) models. In the first part, look at a Walrasian model where we run simple shocks and analyse the results. The second part aims at replicating those shocks with a neo-keynesian model in order to compare results.

1 The Walrasian model

1.1 Getting things ready

We are now changing models, so we need to change the model source files being used. For the Walrasian model, the model file is 02.1-eq.mdl in the src/model/training folder. The data calibration file is 0.2.1-calib.mdl file.

  • Prepare the configuration file.

    • For now we are still running a g shock.
    • Change the project name and the project title to something more relevant
    • Specify the correct calib and model mdl files

  • Adapt the main script to reflect the relevant changes

    • The config script if you created a new one
    • The texdoc function inputs (section 4)
    • The quarto markdown to be rendered for this one is called results_walras_model.qmd

1.2 Launching the simulations

You should now be able to run the simulations. Check that all works correctly with the inputs

1.3 Extend your analysis

Open the results_walras_model.qmd file and program additional graphs or tables in order to see other results

2 The Neo-Keynesian model

2.1 Basic simulations

Replicate what you did for the Walrasian model in the first part with the Neo-Keynesian model

Some information:

  • The model file is called 03.1-eq.mdl and the calibration data file is 03.1-calib.mdl
  • There is no quarto file provided: create a new one based the one you used for the Walrasian model.
  • Changing the project name allows you to keep both the simulation output from the Walrasian model and from the Neo-Keynesian model to allow for comparison later on

3 Going further: create your own shocks

3.1 Create a simple shock from scratch

Using the Neo-Keynasian model, create the following shock: A temporary inflation shock in 2023 set at +6 %.

Some steps:

  • What is the default inflation rate in the steady state?
  • On what exogenous variable should you be applying the shock?
  • Give the scenario a name ie price_shock
  • Create a 2_calib_shock_***.R file in the configuration/scenarii_calib folder (you may use a script known such as 2_calib_shock_g.R as a template)
  • Program the shock in this script
  • Adjust the config script
  • Run the simulation and adjust the .qmd to view more interesting outputs.
